# Item overflow

item overflow are a group of patterns built using
[overflowHandler](?path=/docs/experimental-utilities-overflowhandler--docs)

Patterns have multiple ways of accomplishing a user need and typically use a
combination of components with additional design considerations. The pattern
code we share is meant to serve as an example implementation that can be built
and extended further.

### 1) Set of Tags

The Set of Tags pattern displays a row of tags. If there isnâ€™t enough space to
show all of them, the overflowing tags are grouped in a popover, triggered by an
overflow button labeled +N.

If there are more than 10 overflowing tags, a View all tags button appears in
the popover. Clicking it opens a modal that displays the full list of tags.

This pattern is commonly used to filter or categorize content.

To build this pattern, we recommend including the following components:

- [cds-tag cds-dismissible-tag cds-operational-tag](https://web-components.carbondesignsystem.com/?path=/docs/components-tag--overview)
- [cds-popover](https://web-components.carbondesignsystem.com/?path=/docs/components-popover--overview)
- [cds-modal](https://web-components.carbondesignsystem.com/?path=/docs/components-modal--overview)

and the following utility:

- [overflowHandler](?path=/docs/experimental-utilities-overflowhandler--docs)

These components are used within the `set-of-tags` wrapper component to handle
state management.
