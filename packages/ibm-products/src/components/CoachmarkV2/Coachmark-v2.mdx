## Overview

[Coachmarks](https://pages.github.ibm.com/carbon/ibm-products/components/onboarding/coachmark/usage)
are just-in-time messages used to draw a userâ€™s attention to a particular spot,
and explain new functionality or concepts they might not otherwise notice or
understand.

**Note:**

- The trigger element or first child of `CoachmarkV2` should always be either a
  CoachmarkBeacon or a Button.

## Coachmark code sample

```jsx
<Theme theme={white}>
  <CoachmarkV2
    ariaLabel="CoachmarkV2"
    position={{ x: 151, y: 155 }}
    open={isOpen}
    onClose={handleClose}
    {...args}
  >
    <CoachmarkBeacon
      id="CoachmarkBtn"
      label="Show information"
      onClick={handleBeaconClick}
    ></CoachmarkBeacon>
    <CoachmarkV2.Content highContrast={true}>
      <CoachmarkContent.Header
        closeIconDescription="Close"
        dragIconDescription="Drag"
      ></CoachmarkContent.Header>
      <CoachmarkContent.Body>
        <h2>Hello World</h2>
        <p>this is a description test</p>
        <Button size="sm">Done</Button>
      </CoachmarkContent.Body>
    </CoachmarkV2.Content>
  </CoachmarkV2>
</Theme>
```
