# Add and select patterns

## Overview

The Add and select patterns provide complete, production-ready implementations
for selection workflows in IBM products. These patterns combine Carbon
components with IBM Products components to create cohesive user experiences.

## Available patterns

### MultiAddSelect

A complete pattern for multi-selection workflows with Tearsheet, selection
summary, and success notifications.

**Use when:**

- Users need to select multiple items from a list
- Selection requires a modal/tearsheet interface
- You need real-time selection feedback
- Success confirmation is important

[View MultiAddSelect →](/docs/patterns-add-and-select-multiaddselect--docs)

### SingleAddSelect

A complete pattern for single-selection workflows with hierarchical navigation,
breadcrumbs, and success notifications.

**Use when:**

- Users need to select exactly one item
- Items are organized in a hierarchical structure
- Navigation through categories is required
- Selection requires a modal/tearsheet interface

[View SingleAddSelect →](/docs/patterns-add-and-select-singleaddselect--docs)

## Pattern philosophy

These patterns are **copy-paste ready** - they include everything you need:

✅ **Complete implementations** - Tearsheet, AddSelect, Toast notifications, and
all interaction logic  
✅ **Production-ready** - Fully tested, accessible, and responsive  
✅ **Customizable** - Easy to adapt to your specific use case  
✅ **Well-documented** - Clear API and usage examples

## How to use

1. **Copy the component file** from `example/components/`
2. **Import and use** in your application
3. **Customize props** to match your requirements
4. **Style as needed** using Carbon tokens

## Key differences

| Feature       | MultiAddSelect              | SingleAddSelect               |
| ------------- | --------------------------- | ----------------------------- |
| Selection     | Multiple items (checkboxes) | Single item (radio buttons)   |
| Navigation    | Flat list                   | Hierarchical with breadcrumbs |
| Summary panel | Shows all selected items    | Not applicable                |
| Use case      | Bulk operations             | Category selection            |

## Components used

These patterns are built using:

- **Tearsheet** - Modal container
- **AddSelect** - Core selection component
- **Button** - Action triggers
- **ToastNotification** - Success feedback
- **NoDataEmptyState** - Empty states

## Best practices

1. **Choose the right pattern** - Multi for bulk operations, Single for category
   selection
2. **Customize labels** - Update all text to match your domain language
3. **Handle callbacks** - Always implement `onSubmit` to process selections
4. **Test with real data** - Ensure patterns work with your actual data
   structure
5. **Consider performance** - For large datasets, implement pagination or
   virtualization

## Accessibility

Both patterns include:

- Full keyboard navigation
- ARIA attributes for screen readers
- Focus management
- Semantic HTML structure
- High contrast support
