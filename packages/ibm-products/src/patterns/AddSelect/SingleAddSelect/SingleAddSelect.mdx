## Overview

The SingleAddSelect pattern provides a complete, production-ready implementation
for single-selection workflows with hierarchical navigation. It includes a
Tearsheet container, AddSelect component with breadcrumb navigation, and success
notifications.

> **Copy-paste ready**: The component in
> `example/components/SingleAddSelect/SingleAddSelect.tsx` includes everything
> you need - Tearsheet, breadcrumb navigation, Toast notifications, and all
> interaction logic. Simply copy the file and customize the props.

### Key Features

- **Complete pattern** - Includes Tearsheet, AddSelect, breadcrumb navigation,
  and toast notifications
- **Single selection** - Select exactly one item with radio buttons
- **Hierarchical navigation** - Navigate through nested data structures with
  breadcrumbs
- **Search** - Built-in global search across all levels
- **Breadcrumb navigation** - Clear indication of current location in hierarchy
- **Side panel support** - Optional side panel for displaying item details
- **Success feedback** - Toast notification on successful submission
- **Accessible** - Full keyboard navigation and ARIA support

### Basic usage

```jsx
import { SingleAddSelect } from './components/SingleAddSelect';

const MyComponent = () => {
  const [open, setOpen] = useState(false);

  const items = [
    { id: '1', title: 'Category 1', value: 'cat1' },
    {
      id: '2',
      title: 'Category 2',
      value: 'cat2',
      children: {
        entries: [{ id: '2-1', title: 'Subcategory 1', value: 'subcategory1' }],
      },
    },
  ];

  const handleSubmit = (itemId, value) => {
    console.log('Selected:', itemId, value);
  };

  return (
    <>
      <Button onClick={() => setOpen(true)}>Select category</Button>

      <SingleAddSelect
        open={open}
        setOpen={setOpen}
        items={items}
        onSubmit={handleSubmit}
        title="Select category"
        description="Choose one category from the list below"
      />
    </>
  );
};
```

### Item structure

```typescript
interface AddSelectItem {
  id: string;
  title?: string;
  value?: string;
  subtitle?: string;
  icon?: ReactNode;
  disabled?: boolean;
  meta?: {
    description?: string;
    details?: Record<string, string | number>;
    tags?: string[];
  };
  children?: {
    entries: AddSelectItem[];
  };
}
```

## Hierarchical Navigation

The pattern supports nested data structures with breadcrumb navigation:

- Click items with children to navigate deeper
- Use breadcrumbs to navigate back up the hierarchy
- Search works across all levels of the hierarchy
- Selection is maintained when navigating

## Side Panel Pattern

The `AddSingleItemFromHierarchy` pattern extends the basic hierarchy pattern
with a side panel that displays detailed information about the selected item.
This is useful when:

- Users need additional context to make an informed decision
- Items have rich metadata (descriptions, details, tags)
- The selection involves significant resources or further steps

### When to use the side panel

- **Complex items** - When items have multiple attributes that help with
  selection
- **Need for context** - When users benefit from seeing details before
  committing
- **Rich metadata** - When items have descriptions, specifications, or tags
- **Wide tearsheet** - The pattern uses a wide tearsheet to accommodate the side
  panel

## Best Practices

1. **Keep hierarchies shallow** - Limit to 2-3 levels for better UX
2. **Provide clear labels** - Use descriptive titles that match your domain
3. **Handle submission** - Always provide an `onSubmit` callback
4. **Test navigation** - Ensure breadcrumb navigation works correctly
5. **Consider search** - For deep hierarchies, search becomes essential
6. **Show breadcrumbs** - Help users understand their location in the hierarchy
7. **Use side panel wisely** - Only include it when users need additional
   context
8. **Limit description length** - Keep descriptions to 4 lines maximum for
   readability

## Keyboard Navigation

- **Arrow Up/Down** - Navigate between items
- **Arrow Right** - Navigate to children (if available)
- **Enter/Space** - Select item
- **Ctrl+Home** - Jump to first item
- **Ctrl+End** - Jump to last item
