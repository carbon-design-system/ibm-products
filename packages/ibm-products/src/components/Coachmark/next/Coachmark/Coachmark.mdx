## Overview

[Coachmarks](https://pages.github.ibm.com/carbon/ibm-products/components/onboarding/coachmark/usage)
are just-in-time messages used to draw a user's attention to a particular spot,
and explain new functionality or concepts they might not otherwise notice or
understand.

**Note:**

- The trigger element or first child of `Coachmark` should always be either a
  CoachmarkBeacon or a Button.
- When using an external trigger element (not a child of Coachmark), use the
  `triggerRef` prop to pass a ref to the trigger element. This allows the
  Coachmark to properly manage focus and accessibility attributes.

## Coachmark code sample

```jsx
<Theme theme="white">
  <Coachmark
    position={{ x: 151, y: 155 }}
    open={isOpen}
    onClose={handleClose}
    {...args}
  >
    <CoachmarkBeacon
      id="CoachmarkBtn"
      label="Show information"
      buttonProps={{
        onClick: handleBeaconClick,
        id: 'CoachmarkBtn',
      }}
    />
    <Coachmark.Content>
      <Coachmark.ContentHeader
        closeIconDescription="Close"
        dragIconDescription="Drag"
      />
      <Coachmark.ContentBody>
        <h2>Hello World</h2>
        <p>this is a description test</p>
        <Button size="sm" onClick={handleClose}>
          Done
        </Button>
      </Coachmark.ContentBody>
    </Coachmark.Content>
  </Coachmark>
</Theme>
```

## With external trigger ref example usage

Use the `triggerRef` prop when you need to attach the Coachmark to a trigger
element that exists outside the Coachmark component tree. This is useful when
the trigger button needs to be positioned independently or when integrating with
existing UI elements.
