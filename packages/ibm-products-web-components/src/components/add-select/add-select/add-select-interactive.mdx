## Overview

This interactive example demonstrates a complete, production-ready
implementation of AddSelect using the AddSelectData utility from
`@carbon/ibm-products-primitives`.

The example includes:

- Hierarchical data navigation (countries → states/provinces → cities)
- Real-time search functionality
- Single-select with state management
- Breadcrumb navigation
- Dynamic item counts

## Implementation Steps

Follow these steps to implement AddSelect with the AddSelectData utility in your
web component application:

### Step 1: Install Dependencies

```bash
npm install @carbon/ibm-products-web-components @carbon/ibm-products-primitives
```

### Step 2: Import Required Modules

```typescript
import '@carbon/ibm-products-web-components/es/components/add-select/index.js';
import {
  AddSelectData,
  AddSelectItem,
} from '@carbon/ibm-products-primitives';
```

### Step 3: Define Your Hierarchical Data

Structure your data according to the `AddSelectItem` interface:

```typescript
const hierarchicalData: AddSelectItem[] = [
  {
    id: '1',
    title: 'Parent Item',
    value: 'parent-1',
    children: {
      entries: [
        { id: '1-1', title: 'Child Item 1', value: 'child-1' },
        { id: '1-2', title: 'Child Item 2', value: 'child-2' },
      ],
    },
  },
  {
    id: '2',
    title: 'Another Parent',
    value: 'parent-2',
  },
];
```

### Step 4: Initialize AddSelectData

Create an instance of AddSelectData and set up state management:

```typescript
const dataManager = new AddSelectData();
dataManager.setItems(hierarchicalData);

let currentItems = hierarchicalData;
let selectedItems = new Set<string>();
let searchTerm = '';
let path: Array<{ id: string; title: string }> = [];
let currentParentId = '';
```

### Step 5: Implement Selection Handler

Handle item selection using the utility:

```typescript
const handleItemSelect = (event: CustomEvent) => {
  const { itemId, selected } = event.detail;

  if (selected) {
    // Use exclusive mode for single-select
    dataManager.setSelectedItems(itemId, true);
    selectedItems = new Set([itemId]);

    const selectedItem = dataManager.getItem(itemId);
    console.log('Selected item:', {
      id: selectedItem?.id,
      title: selectedItem?.title,
      value: selectedItem?.value,
    });
  } else {
    dataManager.clearSelections();
    selectedItems = new Set();
    console.log('Selection cleared');
  }

  // Re-render your component
  updateView();
};
```

### Step 6: Implement Navigation Handler

Handle hierarchical navigation:

```typescript
const handleNavigate = (event: CustomEvent) => {
  const { itemId, title } = event.detail;
  const children = dataManager.getItemChildren(itemId);

  if (children.length > 0) {
    currentItems = children;
    currentParentId = itemId;

    // Build breadcrumb path
    const parents = dataManager.getItemParents(itemId);
    path = [
      { id: 'root', title: 'All Items' },
      ...parents.reverse().map((p) => ({ id: p.id, title: p.title || '' })),
      { id: itemId, title },
    ];

    updateView();
  }
};
```

### Step 7: Implement Breadcrumb Handler

Handle breadcrumb navigation:

```typescript
const handleBreadcrumbClick = (event: CustomEvent) => {
  const { index } = event.detail;

  if (index === 0) {
    // Navigate to root
    currentItems = hierarchicalData;
    currentParentId = '';
    path = [];
  } else {
    const targetId = path[index].id;
    const children = dataManager.getItemChildren(targetId);
    currentItems = children;
    currentParentId = targetId;
    path = path.slice(0, index + 1);
  }

  updateView();
};
```

### Step 8: Implement Search Handler

Handle search functionality:

```typescript
const handleSearch = (event: CustomEvent) => {
  const query = event.detail.searchTerm || '';
  searchTerm = query;

  if (query) {
    const results = dataManager.search(query);
    currentItems = results;
    path = [];
  } else {
    // Reset to current level or root
    if (currentParentId) {
      const children = dataManager.getItemChildren(currentParentId);
      currentItems = children;
    } else {
      currentItems = hierarchicalData;
    }
  }

  updateView();
};
```

### Step 9: Render Items with Navigation Indicators

Render items with proper `hasChildren` flags:

```typescript
currentItems.forEach((item) => {
  const hasChildren = dataManager.hasChildren(item.id);
  const isSelected = selectedItems.has(item.id);

  const row = document.createElement('c4p-add-select-row');
  row.setAttribute('item-id', item.id);
  row.setAttribute('title', item.title || '');
  row.setAttribute('subtitle', item.value);
  row.setAttribute('value', item.value || '');
  if (hasChildren) {
    row.setAttribute('has-children', '');
  }
  if (isSelected) {
    row.setAttribute('selected', '');
  }

  row.addEventListener('c4p-add-select-row-select', handleItemSelect);
  row.addEventListener('c4p-add-select-row-navigate', handleNavigate);

  list.appendChild(row);
});
```

### Step 10: Compose the AddSelect Component

Put it all together:

```html
<c4p-add-select>
  <c4p-add-select-body
    items-label="All Items"
    global-search-label="Search"
    global-search-placeholder="Search items..."
    search-results-title="Search results"
  >
    <c4p-add-select-list>
      <!-- Rows added dynamically -->
    </c4p-add-select-list>
  </c4p-add-select-body>
</c4p-add-select>
```

### Step 11: Set Up Event Listeners

Connect event listeners to the body component:

```typescript
const body = document.querySelector('c4p-add-select-body');
body.addEventListener('c4p-add-select-body-search', handleSearch);
body.addEventListener('c4p-add-select-body-breadcrumb-click', handleBreadcrumbClick);

// Update path and itemCount properties
body.path = path;
body.itemCount = currentItems.length;
```

## AddSelectData Utility

The AddSelectData utility provides powerful methods for managing hierarchical
data:

### Core Methods

- **`setItems(items)`** - Initialize or replace data
- **`getItems()`** - Get all items
- **`getItem(id)`** - Get a specific item by ID

### Selection Methods

- **`setSelectedItems(ids, exclusive)`** - Select items (exclusive for
  single-select)
- **`getSelectedItems()`** - Get all selected items
- **`isSelected(id)`** - Check if an item is selected
- **`clearSelections()`** - Clear all selections

### Hierarchy Methods

- **`getItemChildren(id)`** - Get direct children
- **`getItemParent(id)`** - Get parent item
- **`getItemParents(id)`** - Get all ancestors
- **`hasChildren(id)`** - Check if item has children
- **`getItemDescendants(id)`** - Get all descendants
- **`getItemDepth(id)`** - Get hierarchy depth

### Search and Sort

- **`search(query, options)`** - Search items
- **`sort(compareFn, recursive)`** - Sort items

[View full AddSelectData documentation →](https://github.com/carbon-design-system/ibm-products/blob/main/packages/ibm-products-primitives/src/add-select/add-select-data.md)

## Complete Example

The interactive story above demonstrates all these concepts working together.
Key features:

- **Hierarchical Navigation**: Click items with children to drill down through
  the hierarchy
- **Breadcrumb Navigation**: Click breadcrumbs to navigate back up
- **Search**: Type to search across all items in the hierarchy
- **Single-select**: Select one item at a time (use `multi` attribute for multi-select)
- **State Management**: All state managed through AddSelectData utility
- **Console Logging**: Selected items logged for debugging

## Events

The AddSelect web components emit the following custom events:

### c4p-add-select-row-select

Fired when a row is selected or deselected.

```typescript
event.detail = {
  itemId: string,
  selected: boolean
}
```

### c4p-add-select-row-navigate

Fired when navigating to an item's children.

```typescript
event.detail = {
  itemId: string,
  title: string
}
```

### c4p-add-select-body-search

Fired when the search term changes.

```typescript
event.detail = {
  searchTerm: string
}
```

### c4p-add-select-body-breadcrumb-click

Fired when a breadcrumb is clicked.

```typescript
event.detail = {
  index: number
}
```
