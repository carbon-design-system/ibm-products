## Overview

AddSelect is a composable component system for building flexible add/select
interfaces. It provides a component-based approach where you compose the UI
using smaller, focused components instead of a monolithic component. This design
enables maximum flexibility while maintaining consistency and accessibility.

The implementation uses the compound component pattern in React, with distinct
and well-defined sections. You need to import the main
[`AddSelect`](./AddSelect.tsx) component, and all other subcomponents are
available as static properties.

### Key Features

- **Composable Architecture**: Build complex selection interfaces by combining
  simple, focused components
- **Multi-select & Single-select**: Support for both checkbox-based
  multi-selection and radio button-based single selection
- **Hierarchical Navigation**: Built-in support for navigating through nested
  item structures
- **Search & Filter**: Global and column-level search with custom filter actions
- **Selection Summary**: Display and manage selected items with optional
  accordion view
- **Item Details Panel**: Show detailed information about individual items
- **Keyboard Navigation**: Full keyboard support for accessibility
- **Flexible Layouts**: Vertical and horizontal layouts for different use cases

## Component Hierarchy

The AddSelect component system follows a clear hierarchical structure:

```
AddSelect (Root Container)
├── AddSelect.Body (Main Content Area)
│   ├── Header Section (Search, Breadcrumbs, Item Count)
│   └── AddSelect.Content (Items Container)
│       ├── AddSelect.Column (Optional Column Wrapper)
│       │   ├── Column Header (Search, Title, Select All)
│       │   └── AddSelect.Row (Individual Items)
│       └── AddSelect.Row (Direct Items, no column)
│
├── AddSelect.SelectionSummary (Selected Items Panel)
│   └── AddSelect.SelectionSummaryItem (Individual Selected Item)
│
└── AddSelect.ItemPanel (Item Details Panel)
```

### Component Flow

1. **AddSelect** - Root container that manages selection state
2. **AddSelect.Body** - Contains search, filters, and item list
3. **AddSelect.Content** - Handles keyboard navigation and layout
4. **AddSelect.Column** (Optional) - Groups items with column-specific features
5. **AddSelect.Row** - Individual selectable items
6. **AddSelect.SelectionSummary** (Optional) - Shows selected items
7. **AddSelect.SelectionSummaryItem** (Optional) - Individual selected item
   display
8. **AddSelect.ItemPanel** (Optional) - Detailed item information

## Subcomponents

### AddSelect (Root)

The main container component that provides context for selection state and
callbacks. All child components access selection state through this context.

**When to use**: Always required as the root wrapper for any AddSelect
interface.

### AddSelect.Body

Contains the header section (search, breadcrumbs, item count) and the main
content area. Handles global search and navigation.

**When to use**: Required for displaying the main item selection interface.

### AddSelect.Content

Container for items with built-in keyboard navigation support. Manages focus and
provides grid-based navigation.

**When to use**: Required as a wrapper for AddSelect.Row or AddSelect.Column
components.

### AddSelect.Column

Optional column wrapper that provides column-specific features like column
title, column-level search, and "Select All" functionality.

**When to use**:

- When organizing items into distinct columns
- When you need column-level search or "Select All"
- For hierarchical navigation with multiple levels

### AddSelect.Row

Individual selectable item with support for checkboxes (multi-select) or radio
buttons (single-select).

**When to use**: For every selectable item in your interface.

### AddSelect.SelectionSummary

Displays a list of selected items with optional count badge and edit
functionality.

**When to use**:

- When you need to show selected items separately
- For review/confirmation workflows
- When building multi-step selection processes

### AddSelect.SelectionSummaryItem

Individual item display within the selection summary, with optional accordion
view and remove functionality.

**When to use**:

- For custom rendering of selected items
- When you need accordion-style item details
- When items need individual remove buttons

### AddSelect.ItemPanel

Side panel that displays detailed information about a specific item.

**When to use**:

- When items have additional details to display
- For preview/inspection workflows
- When users need more context before selecting

## Usage Patterns

### Pattern 1: Single Select

Simple list with radio button for selection.

```jsx
const [selectedItem, setSelectedItem] = useState(null);

<AddSelect selectedItems={selectedItems} onItemSelect={handleItemSelect}>
  <AddSelect.Body itemsLabel="Items" itemCount={items.length}>
    <AddSelect.Content>
      {items.map((item) => (
        <AddSelect.Row
          key={item.id}
          itemId={item.id}
          title={item.title}
          subtitle={item.subtitle}
          value={item.value}
        />
      ))}
    </AddSelect.Content>
  </AddSelect.Body>
</AddSelect>;
```

### Pattern 1: Basic Multi-Select List

Simple list with checkboxes for multiple selection. Use the `multi` prop on
AddSelect.Column to control selection mode.

```jsx
<AddSelect selectedItems={selectedItems} onItemSelect={handleItemSelect}>
  <AddSelect.Body itemsLabel="Items" itemCount={items.length}>
    <AddSelect.Content>
      <AddSelect.Column multi={false}>
        {items.map((item) => (
          <AddSelect.Row key={item.id} {...item} />
        ))}
      </AddSelect.Column>
    </AddSelect.Content>
  </AddSelect.Body>
</AddSelect>
```

### Pattern 3: With Search and Filters

Add global search and custom filter actions.

```jsx
<AddSelect.Body
  itemsLabel="Items"
  globalSearchLabel="Search"
  globalSearchPlaceholder="Search items"
  itemCount={filteredItems.length}
  onSearch={setSearchTerm}
  actionsSlot={
    <Dropdown
      id="filter"
      label="Filter"
      items={filterOptions}
      onChange={handleFilterChange}
    />
  }
>
  <AddSelect.Content>
    {filteredItems.map((item) => (
      <AddSelect.Row key={item.id} {...item} />
    ))}
  </AddSelect.Content>
</AddSelect.Body>
```

### Pattern 4: With Columns and Select All

Organize items into columns with "Select All" functionality.

```jsx
<AddSelect.Content layout="horizontal">
  <AddSelect.Column
    title="Available items"
    showSelectAll
    multi
    itemCount={items.length}
    allSelected={allSelected}
    onSelectAll={handleSelectAll}
  >
    {items.map((item) => (
      <AddSelect.Row key={item.id} {...item} />
    ))}
  </AddSelect.Column>
</AddSelect.Content>
```

### Pattern 5: With Selection Summary

Display selected items in a separate panel.

```jsx
<AddSelect selectedItems={selectedItems} onItemSelect={handleSelect}>
  <AddSelect.Body itemsLabel="Items" itemCount={items.length}>
    <AddSelect.Content>
      {items.map((item) => (
        <AddSelect.Row key={item.id} {...item} />
      ))}
    </AddSelect.Content>
  </AddSelect.Body>

  <AddSelect.SelectionSummary
    title="Selected items"
    selectedItems={selectedItemsArray}
    showCount
    showEditIcon
    onEdit={handleEdit}
  >
    {selectedItemsArray.map((item) => (
      <AddSelect.SelectionSummaryItem
        key={item.id}
        item={item}
        useAccordion
        onRemove={handleRemove}
      />
    ))}
  </AddSelect.SelectionSummary>
</AddSelect>
```

### Pattern 6: With Item Details Panel

Show detailed information about items.

```jsx
<AddSelect.Row
  itemId={item.id}
  title={item.title}
  value={item.value}
  hasItemPanel
  onItemPanelClick={handleItemPanelClick}
  itemPanelOpen={itemPanelOpen && selectedItem?.id === item.id}
/>;

{
  itemPanelOpen && (
    <AddSelect.ItemPanel
      title="Item details"
      item={selectedItem}
      onClose={() => setItemPanelOpen(false)}
    />
  );
}
```

### Pattern 7: Hierarchical Navigation

Navigate through nested item structures.

```jsx
<AddSelect.Content layout="horizontal">
  <AddSelect.Column title={currentLevel.title} onNavigate={handleNavigate}>
    {currentLevel.items.map((item) => (
      <AddSelect.Row
        key={item.id}
        {...item}
        hasChildren={item.children?.length > 0}
      />
    ))}
  </AddSelect.Column>
</AddSelect.Content>
```
